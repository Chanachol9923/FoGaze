An End-to-End Assistive Robotics Pipeline

Target Object Specification using Gaze-Driven Stereo Vision and Spatial Relationship Mapping for People with Physical Disabilities

01 · Research Target

What this research is for

Target Group

People with severe physical disabilities
(ALS, paralysis)

The Goal

Turn “looking” into a “command to fetch an object”

Core Concept: Look → Lock the object → Robot fetches it
02 · Core Objectives

Four core objectives

1

Gaze-Driven

Compute gaze coordinates with an ordinary webcam

2

Stereo Vision

Locate the position & depth (3D space) of objects

3

Spatial Mapping

Turn coordinates into spoken descriptions of the surroundings

4

Assistive Automation

Trigger pick-and-place hands-free

Runs in real time on an ordinary CPU — no special-purpose hardware required.

03 · The Problem

Existing tools don't fit this user

Current Limits

  • Gaze-tracking IR hardware is very expensive
  • Often head-mounted → intrusive for the patient
  • Tracks gaze on a “flat screen”, not “real 3-D objects”

Our Solution

Low-cost · No wearables · Runs on a normal CPU

04 · System Architecture

Dual-Camera Pipeline — two cameras, one frame

The Logic: Gaze coordinates ∩ YOLO object box = the object the user selected

05 · Technical Stack

Technology stack — open-source · CPU-friendly

Frontend UI

Python 3 + Dear ImGui
lightweight & fast

AI Perception

MediaPipe Face Landmarker + EyeTrax ML Model
Ultralytics YOLOv8 (Object Detection)

Robotics Backend

ROS 2 Humble + MoveIt
arm trajectory planning

Python 3 on Ubuntu 22.04 · the robot layer is optional and degrades gracefully if absent.

06 · Core Features

Key features

Gaze Smoothing

Kalman + EMA filters reduce jitter and lock the target steady

Face Lock

Auto-zoom on the face → sharper eye coordinates

Triple-Blink Trigger

Blink 3× = click to select (hands-free)

Spatial Mapping

Turn position into speech

“The cup is on your right.”
07 · Robot Integration

Look · Blink · Grab

Collision Avoidance

Nearby objects you didn't look at become automatic obstacles

Safe Planning

The arm plans a collision-free path to reach the target safely

Grasp Analysis

Compute object size so the gripper grips firmly — no drops

Unselected objects = obstacle boxes (red dashed) the arm must avoid.
08 · Conclusion

An assistive robot, controlled by the eyes

Key Success

Control an assistive robot for people with disabilities using gaze — running on a CPU in real time

Next Step
